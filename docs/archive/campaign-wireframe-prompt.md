# 캠페인 탭 — Claude Design Wireframe 프롬프트

> **📦 보관 문서 (스크래치):** 캠페인 탭 와이어프레임 생성용 일회성 프롬프트. 캠페인 기능은 이후
> 실제로 구현되었다(`klow_server/src/modules/campaigns/`, klow_brand `(authed)/campaigns/`) —
> 현행 스펙은 [`../server/modules/campaigns.md`](../server/modules/campaigns.md) 참고.

> klow_brand "캠페인" 기능의 와이어프레임 생성용. Claude Design의 **Wireframe 템플릿**에 아래 코드블록을 그대로 붙여넣으세요.

## 결정 사항 요약

| 항목 | 결정 |
|------|------|
| 배치 | 상단 pill 네비 **최상위 메뉴** (`/campaigns` 독립 페이지) |
| 캠페인 구조 | **다중 캠페인** — 캠페인마다 여러 인플루언서 링크 발급 |
| 발급 대상 | 인플루언서 **이름/SNS 핸들 직접 입력** |
| "링크 노출도" | 인플루언서 SNS 실측 불가 → **링크 클릭/유입 수로 대체** |

## ✂️ 복사용 프롬프트

```
[제품 맥락]
K-뷰티 브랜드의 셀프서비스 대시보드(모바일 우선 반응형 웹, 전부 한국어)에 새로 추가할 "캠페인" 기능의 와이어프레임을 만들어줘. 사용자는 브랜드 운영자다. 최종 비주얼이 아니라 레이아웃·정보 위계·화면 흐름(UX)을 잡는 와이어프레임이 목표다.

[기능 한 줄 정의]
브랜드가 자신의 "브랜드관(스토어 홈)" 링크를 인플루언서에게 발급 → 인플루언서가 자기 SNS 링크인바이오에 게시 → 그 링크로 들어온 방문자의 유입수와 구매 전환을 브랜드가 추적한다.

[전역 레이아웃]
- 상단에 가로 pill 형태의 sticky 네비게이션 바가 있다: 로고 | 홈 | 크리에이터 찾기 | 시딩 & 링크 | 캠페인 | 설정. "캠페인"이 현재 활성 탭.
- 본문은 중앙 정렬 단일 컬럼 컨테이너. 모바일 우선이지만 데스크탑에서도 보기 좋게.
- 리스트는 표보다 카드 row(행) 스타일 선호. 섹션은 카드 컨테이너로 묶는다.

다음 4개 화면을 만들어줘:

[화면 1] 캠페인 목록 (캠페인 탭 진입 시 기본 화면)
- 페이지 헤더: "캠페인" 제목 + 우측에 "+ 새 캠페인" 버튼.
- 상단 요약 KPI 카드 4개(가로 배치): 진행 중 캠페인 수 / 총 유입수(클릭) / 총 구매수 / 총 매출.
- 캠페인 카드 리스트(세로). 각 카드: 캠페인 이름, 기간(시작~종료), 상태 뱃지(진행중/종료), 발급 링크 수(인플루언서 N명), 핵심 지표 3개(유입수·구매수·전환율). 카드 클릭 시 상세로 이동.
- 데이터가 없을 때 빈 상태(empty state): 안내 문구 + "+ 첫 캠페인 만들기" CTA.

[화면 2] 새 캠페인 생성 (모달)
- 모달 폼: 캠페인 이름(필수), 설명(선택), 기간(시작일/종료일, 선택). 대상 브랜드관은 본인 것으로 자동 지정되어 읽기전용 안내.
- 하단: 취소 / 생성하기 버튼. 생성 후 해당 캠페인 상세로 이동.

[화면 3] 캠페인 상세 (가장 중요한 화면)
- 상단: 뒤로가기 + 캠페인 이름, 기간, 상태 뱃지.
- 캠페인 요약 지표 바: 총 유입수(클릭) / 총 구매수 / 전환율 / 총 매출.
- "인플루언서 링크" 섹션 헤더 + 우측 "+ 인플루언서 링크 발급" 버튼.
- 정렬/필터 컨트롤(예: 유입 많은 순, 구매 많은 순, 최신 발급순).
- 인플루언서 링크 목록을 행(row) 리스트로. 각 행에 한눈에:
  · 인플루언서 이름 + SNS 핸들/플랫폼 아이콘
  · 발급된 링크(짧게 표시) + "복사" 버튼
  · 발급/노출 상태 뱃지(예: 발급됨 → 유입발생 → 구매발생 단계)
  · 유입수(클릭)  · 구매수  · 매출(또는 전환율)
  · 발급일
  · 행 클릭 시 인플루언서별 상세(화면 4 또는 펼침)
- 인플루언서가 한 명도 없을 때 빈 상태 + 발급 CTA.

[화면 4] 인플루언서 링크 발급 (모달)
- 폼: 인플루언서 이름(필수), SNS 플랫폼 선택(인스타/유튜브/틱톡 등), SNS 핸들(필수), 메모(선택).
- 발급하기 누르면 고유 링크가 생성되어 모달 안에 표시 + 큰 "링크 복사" 버튼(인플루언서에게 전달용). 안내문: "이 링크를 인플루언서에게 전달하면, 인플루언서가 자기 SNS 프로필(링크인바이오)에 게시합니다."

[추가로 보여주면 좋은 것]
- 화면 3의 행을 펼치거나 화면 4 형태로, 인플루언서 한 명의 상세: 기간별 유입 추이(간단한 라인/바 차트 자리표시), 누적 유입·구매·매출, 발급 링크 정보, 복사 버튼.

[톤] 와이어프레임이므로 색·이미지는 최소화하고 구조/위계/플로우에 집중. 한국어 라벨 사용. 컴포넌트는 카드 row, KPI 카드, 상태 뱃지, 모달, 복사 버튼 중심.
```

## 참고 — 구현 단계로 미룬 결정

- 전환/구매 attribution 기준(마지막 클릭 vs 전체, 추적 윈도우)
- 링크 식별 방식(`?ref=` 쿼리 vs 단축 경로) + 쿠키/세션 추적
- 캠페인 종료/아카이브, 링크 만료 정책
- 매출 통화 표기(klow 결제는 USD 기준)
